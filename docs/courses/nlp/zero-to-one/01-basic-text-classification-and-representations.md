# Basic Text Classification and Representations / 基础文本分类与表示

Covered in: Week 1 (`2-NeuralClassifier`)

文本分类是课程里最简单的神经 NLP 模板。它的重要性不在于模型本身有多强，而在于后面很多模型都可以视为在这个模板上不断替换“表示层”和“序列建模层”。

## 最基础的任务形式

最简单的设定是：

$$
S = (x_1, x_2, \dots, x_T) \rightarrow y
$$

这里 `S` 是输入序列，`y` 是输出标签。

情感分类是最典型例子：

- 输入：一段影评
- 输出：positive 或 negative

这个形式看起来简单，但已经包含了后续所有任务的雏形：

- 输入是一串离散符号
- 模型需要把它们变成内部表示
- 最后要输出一个有意义的预测

后面更复杂的任务，只是在这里把输出空间从“标签集合”换成了“词表”或“另一个序列”。

## 最基础的 NLP pipeline

一个简单神经分类器通常包含四步：

1. 定义词表 `V`
2. 把词映射成向量
3. 把多个词向量压成序列表示
4. 用预测层输出类别分布

压缩成一句话就是：

`word ids -> embeddings -> sequence representation -> prediction`

如果再细一点，可以写成：

1. 文本先被切成 `tokens`
2. 每个 token 映射成向量
3. 多个 token 向量组合成句子表示
4. 句子表示送进预测层
5. 预测结果与真实标签比较
6. 通过损失函数更新模型参数

最简单的神经 NLP 系统已经是一个端到端的学习过程。

## 词表与 `<UNK>`

自然语言里会出现大量稀有词、拼写错误、emoji、多语言混杂和特殊符号。

因此实际建模时通常不会把所有可能出现的词都放进词表，而是：

- 先定义一个受控的 `vocabulary`
- 未登录词统一映射成 `<UNK>`

这是一种工程近似，优点是实现简单，缺点是大量信息会在词表外被压缩到同一个符号里。

这里要特别注意一点：

- 词表不是语言本身
- 词表只是模型当前能直接分配参数的离散单元集合

这也是后面 `tokenization` 会重新成为课程重点的原因。

## one-hot、bag-of-words、TF-IDF 为什么不够

这些传统表示法实现简单，但有共同缺点：

- 维度高
- 稀疏
- 不表达相似性
- 不同词之间几乎没有参数共享

例如 `good` 和 `great` 在 `one-hot` 空间里和陌生词一样彼此正交。

`bag-of-words` 和 `TF-IDF` 作为基线模型并不是没有价值，它们的价值主要在于：

- 容易实现
- 训练成本低
- 在小数据上有时也能给出不错结果

但从课程主线看，它们最主要的局限有两类：

- 不能自然表达语义接近性
- 对词序和结构几乎不敏感

## embedding lookup 在做什么

神经模型通常不会直接使用稀疏向量，而是从一个共享矩阵里查稠密向量：

$$
E \in \mathbb{R}^{|V| \times d}
$$

这里：

- `|V|` 是词表大小
- `d` 是向量维度

词表中的每个词对应矩阵里的一行。

这一步常被叫作 `lookup`，因为它本质上确实像查表：

- 输入是词 ID
- 输出是对应那一行向量

但它不是死表，而是训练中会不断更新的参数矩阵。

因此 `embedding` 不是“预处理结果”，而是模型的一部分。

## 为什么 embedding 会带来泛化

如果两个词在训练中经常出现在相似上下文里，那么它们的梯度更新方向往往也会相似。

长期下来，模型会倾向于把这些词推到向量空间中更接近的位置。

这就带来一个很重要的效果：

- 即使训练样本没见过完全相同的句子
- 模型也可能通过相似词的参数共享做出合理泛化

这正是神经表示法相对稀疏表示的关键优势。

## 序列表示最简单的组合方式

最简单的序列表示方法是 pooling，例如：

- `sum pooling`
- `average pooling`
- `max pooling`

这些做法便于实现，但会带来明显信息损失：

- 词序容易被弱化
- 句法结构几乎没有被显式建模
- 长距离关系很难表示

这也是后面必须引入 `RNN` 和 `attention` 的原因。

这里可以把三种 pooling 的差异先粗略记住：

### sum pooling

- 把所有词向量相加
- 长句和短句的向量尺度可能差很多

### average pooling

- 先相加再除以长度
- 比 `sum pooling` 更不受句长影响

### max pooling

- 每一维取最大值
- 更像在抓最强激活特征

它们都能做出一个句子表示，但共同弱点是：句内结构关系表达得比较粗。

## 为什么说这类模型“更像连续版词袋”

因为如果模型只做简单 pooling，那么它本质上还是在回答：

- 这句话里都出现了哪些词
- 哪些词更重要

它很难稳定回答：

- 这些词之间是什么关系
- 谁修饰谁
- 顺序变化有没有改变含义

因此“词向量 + pooling”虽然已经比传统词袋强，但依然只能算课程里的起点。

## 预测层与损失函数

如果是二分类，常见输出形式是 `sigmoid`；
如果是多分类，常见输出形式是 `softmax`。

训练时通常使用 `cross-entropy loss`。
它的直观含义是：正确标签的预测概率越高越好。

如果更具体一点：

- 二分类时，模型通常预测一个介于 0 和 1 之间的概率
- 多分类时，模型通常给所有类别分配一个概率分布

训练目标不是“猜对就行”，而是希望：

- 正确类别概率尽量高
- 错误类别概率尽量低

所以 `cross-entropy` 的本质是一个概率分布层面的训练目标。

## 参数到底在学什么

这个基础模型里，主要可学习参数通常包括：

- `embedding matrix`
- 组合层参数（如果有）
- 输出层参数

其中最重要的认知是：

- 训练学到的不只是分类器
- 还包括词表示本身

`embedding` 会随着任务监督一起被塑形。

## 为什么这类模型适合作为课程起点

因为它已经具备了现代 NLP 的几个核心思想：

- 用连续表示代替离散符号
- 用统一的向量空间做特征抽取
- 用端到端训练同时学表示和预测

后面很多更复杂的模型，本质上都还保留着这些思想，只是把“句子如何表示”这一步做得更强了。

## 这一节的核心限制

这类基础模型已经建立了现代 NLP 的基本模板，但仍然很弱：

- 无法可靠建模词序
- 基本不处理语法结构
- 长距离依赖很弱
- 每个词对整体表示的贡献方式较粗糙

## 需要记住的结论

- 文本分类最基础的结构是 `token -> embedding -> sequence representation -> prediction`。
- `embedding` 是从离散词到连续表示的关键一步。
- 简单 pooling 模型适合入门，但不足以支撑更复杂的语言现象。
- 这类模型已经体现了端到端学习，但还没有真正解决序列建模问题。
