# Course Overview and NLP Basics / 课程总览与 NLP 基础

Covered in: Week 1 (`1-Intro`)

这篇说明课程范围，以及自然语言处理的基本困难。重点是先建立问题空间和课程主线，不展开具体模型公式。

## NLP 在解决什么问题

`NLP` 的目标是让机器处理自然语言。

课程开头给出的典型任务可以分成两类：

- 理解类任务：sentiment analysis、information extraction、question answering
- 生成类任务：machine translation、summarization、chatbots

它们表面形式不同，但都涉及两个基础问题：

- 文本怎么表示
- 序列怎么建模

后面的内容基本都围绕这两个问题展开。

如果把这些任务再抽象一层，可以看到它们只是输入输出形式不同：

- 文本到标签：sentiment analysis、topic classification
- 文本到文本：translation、summarization、dialogue
- 文本到结构：information extraction、parsing、slot filling

无论是哪一种，模型都要先解决一个共同问题：离散的语言符号如何进入可计算的表示空间。

## 为什么自然语言难

自然语言的难点不只是词表大，更重要的是歧义。

课件开头强调了几类常见歧义：

- `syntactic ambiguity`：同一句话可以有不同句法结构
- `semantic ambiguity`：同一句话在不同语境里可以有不同含义
- `pragmatic ambiguity`：需要常识和上下文才能确定真实意图

人类在阅读时通常会自动消解这些歧义，模型则需要显式建模。

语言建模不是简单的模式匹配。模型至少要面对下面几类困难：

- 同一个词可能有多个词义
- 同一句话可能有多个合法结构分析
- 前文很远的一个词，可能决定后文解释
- 真实世界知识常常会影响语言理解

例如在日常阅读里，人类会自然利用：

- 词义常识
- 句法常识
- 说话场景
- 世界知识

而模型在早期并没有这些现成能力，所以课程才需要先从最基础的表示学习和序列建模讲起。

## 为什么“语言难”会直接影响模型设计

课程后面每一代方法，其实都在处理这里提到的某一个困难：

- `embedding` 解决的是“词与词完全彼此孤立”这个问题
- `n`-gram 解决的是“如何给序列分配概率”这个问题
- `RNN` 解决的是“固定窗口不够看”这个问题
- `LSTM / GRU` 解决的是“理论能看远，训练学不会”这个问题
- `attention` 解决的是“所有输入信息被压成一个固定向量”这个问题
- `transformer` 解决的是“循环结构串行、路径长”这个问题
- `pretraining` 解决的是“有标签数据太少”这个问题
- `post-training` 解决的是“会续写不等于会按用户意图工作”这个问题

## 从课程主线看方法演化

课程主线可以压缩成下面这条链：

1. 先从最简单的文本分类和词表示开始
2. 再讨论如何给整段文本分配概率，也就是语言模型
3. 然后引入 `RNN / LSTM / GRU` 处理序列和长距离依赖
4. 再通过 `attention` 和 `transformer` 提高建模能力和并行性
5. 再进入 `pretraining` 和 `transfer learning`
6. 最后讨论数据、评测、文本生成和现代大模型后训练

这条主线需要始终保留。后面的每一章都不是孤立出现的，而是在补前一代方法的不足。

更具体一点，可以把它看成三段：

### 第一段：如何表示文本

这一段解决的问题是：

- 词如何表示
- 序列如何压缩
- 句子概率如何建模

对应内容主要是：

- one-hot / bag-of-words
- embeddings
- `n`-gram language models
- neural language models

### 第二段：如何建模长上下文和复杂依赖

这一段解决的问题是：

- 固定窗口为什么不够
- 长距离依赖如何传递
- 编码器和解码器如何建立更灵活的信息流

对应内容主要是：

- `RNN`
- `LSTM / GRU`
- `seq2seq + attention`
- `transformer`

### 第三段：如何把模型做大、做通用、做可用

这一段解决的问题是：

- 如何用无标签语料先学通用能力
- 如何迁移到下游任务
- 如何构造数据和评测
- 如何让模型按指令工作

对应内容主要是：

- `pretraining`
- `transfer learning`
- data annotation / evaluation
- text generation
- in-context learning / post-training

## 课程里的核心问题

后续内容基本都在回答下面这些问题：

- 词应该表示成什么
- 句子和段落应该如何压缩成有用表示
- 模型怎么利用上下文
- 固定窗口为什么不够
- 长距离依赖怎么处理
- 怎么让模型同时看左右上下文
- 怎么用大量无标签文本先学到通用能力
- 怎么判断一个模型分数高是不是真的更强
- 怎么让模型输出更自然、更有用、更听指令

这组问题里，前半部分更偏“建模能力”，后半部分更偏“系统能力”。

- 建模能力关注的是：模型能不能表示、理解、预测序列
- 系统能力关注的是：模型在真实任务里是否有用、是否稳定、是否可信

课程后半段因此会从模型公式转向数据、评测和对齐。

## 课程中最容易产生的误解

第一次学这门课时，最常见的误解通常有三种。

### 误解 1：后面的模型只是“更复杂的公式”

实际上不是。
后面的方法通常是在解决前一代方法的明确缺陷，而不是单纯把公式变长。

### 误解 2：所有 NLP 问题都完全不同

也不是。
很多任务最后都能写成统一的“输入表示 + 条件建模 + 输出分布”。

### 误解 3：模型分数高就等于真的更强

课程后半段会专门说明为什么这个判断经常不成立。

## 读这门课时建议一直追问的三个问题

每遇到一个新模型，都可以问自己：

1. 它在解决上一代方法的什么问题？
2. 它引入了什么新的假设或代价？
3. 它为什么还不够，后面为什么还要继续升级？

如果这三个问题能回答清楚，课程主线基本就不会断。

## 这一篇需要记住的结论

- `NLP` 同时包含理解和生成两类任务。
- 语言的主要难点来自歧义、上下文依赖和表达不规则。
- 这门课不是零散地介绍模型，而是在讲一条方法升级链。
- 后面的模型演化不是“越来越复杂”，而是在逐步修补前一代方法的结构缺陷。
