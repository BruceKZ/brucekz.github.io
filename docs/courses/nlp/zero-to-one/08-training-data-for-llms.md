# Training Data for LLMs / 大模型训练数据

Covered in: Week 4 guest lecture

这一节补的是原始 `notes/` 里缺失的一块：大模型能力不只来自架构，也高度依赖训练数据的构成和清洗方式。

## 为什么训练数据本身是课程内容

对大模型而言，数据不是后勤问题，而是能力来源的一部分。

同样的模型结构，如果训练数据差异很大，最终表现也可能差很多。

现代大模型的发展很容易让人误以为“架构是最关键的”，但课程想强调的是：

- 架构决定模型能学什么
- 数据决定模型实际学到了什么

没有合适的数据设计，再强的架构也可能学出很差的能力结构。

## 训练数据通常分成哪几层

可以粗略分成三类：

### 预训练数据

- 大规模原始文本
- 目标是学习语言分布、共现结构和广泛知识

这一层更多决定：

- 语言覆盖面
- 基础知识范围
- 文本风格分布
- 长上下文与短上下文的统计结构

### instruction data

- 形式通常是 `instruction -> ideal answer`
- 目标是让模型更像助手，而不是纯续写器

这一层更多决定：

- 模型是否会按要求回答
- 是否会遵守输出格式
- 是否会更像“任务执行者”而不是“自由续写器”

### preference data

- 形式通常是多个回答之间的人类偏好比较
- 目标是为后续偏好优化提供监督

这一层更多决定：

- 多个合理回答中，模型更偏向哪一种风格
- 模型输出是否更接近“有帮助、无害、符合用户偏好”

## 预训练数据为什么不能直接“抓网页就用”

原始网页数据通常存在很多问题：

- 重复样本很多
- 噪声多
- 质量差异很大
- 领域分布不稳定
- 可能混入 benchmark 或测试集污染

因此常见的数据处理步骤包括：

- `filtering`
- `deduplication`
- `curation`
- mixture design

“抓网页”最多只是起点，不是训练数据工程的终点。

真正影响模型质量的是后面一整套筛选和混合过程。

## deduplication

`deduplication` 的作用是去重。

去重的意义不只是节省存储，更重要的是避免：

- 模型把重复样本记得过于牢固
- 某些来源在训练中被异常放大
- benchmark contamination 风险升高

如果不做去重，模型可能会：

- 在某些文本模式上过拟合
- 把重复内容学成不成比例的重要知识
- 在评测里看起来比真实泛化能力更强

因此去重既影响训练稳定性，也影响评测可信度。

## filtering

过滤的目标是提高平均数据质量。

可能的过滤标准包括：

- 语言识别
- 文本长度
- 噪声比例
- 格式异常
- 低质量网页内容

过滤本质上是在做一个判断：

- 哪些文本值得模型花参数容量去学

因为模型容量和训练预算都是有限的，低质量数据过多时，模型会把大量能力浪费在噪声模式上。

## curation 与 mixture

现代大模型往往不是用单一来源数据训练，而是混合多个来源。

因此要考虑：

- 不同来源占比
- 不同领域覆盖
- 不同风格文本的配比

这会直接影响模型的知识分布、写作风格和迁移能力。

更直白地说，数据混合比例会塑造模型的“性格”和“擅长领域”。

例如不同来源的比例变化，可能影响：

- 学术文本能力
- 网页常识能力
- 代码能力
- 对话能力
- 安全与风格偏好

因此 mixture design 并不是数据整理小事，而是模型能力设计的一部分。

## contamination

`contamination` 指训练数据和评测数据之间发生泄漏。

如果 benchmark 样本被模型在训练中见过，那么：

- 评测分数会被高估
- 结果不能真实反映泛化能力

因此训练数据治理和评测设计是连在一起的。

课程后面讲评测时，会反复强调 benchmark 结果不能脱离数据来源和数据处理流程单独理解。

## instruction data 与 preference data 的作用差异

它们都发生在预训练之后，但目标不同：

- `instruction data` 主要让模型学会“按指令回答”
- `preference data` 主要让模型学会“在多个合理回答里更接近人类偏好”

这两层不能混为一谈。

一个模型可能已经：

- 能听懂指令

但还未必：

- 在多个回答里稳定选择人类更偏好的那个

因此 instruction tuning 和 preference optimization 常常是两个不同阶段。

## 为什么这节内容和后训练直接相关

大模型能力可以粗略分成两层：

- 预训练数据决定底层知识和语言建模能力
- instruction / preference 数据决定交互方式和行为风格

因此如果后面要理解：

- instruction tuning
- RLHF
- DPO

那么先把训练数据分层搞清楚会更容易。

## 需要记住的结论

- 大模型能力高度依赖训练数据构成。
- 数据处理通常至少包括 `filtering`、`deduplication` 和 `curation`。
- instruction data 和 preference data 对应的是不同阶段的能力塑形。
- `contamination` 会直接破坏评测可信度。
- 训练数据设计本身就是模型能力设计的一部分。
