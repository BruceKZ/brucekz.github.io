# Text Generation / 文本生成

Covered in: Week 6 (`19+20-Text_Generation_part1`)

这一部分的主线是：语言模型学会预测下一个 token，并不自动等于它能生成高质量文本。生成时还涉及解码策略、训练目标错位和评测问题。

## 训练目标与生成目标不一致

大多数生成模型的基础训练目标是 `maximum likelihood estimation`。

训练时模型看到的历史通常是真实文本；
生成时模型必须不断把自己的输出再喂回去。

这会带来两类问题：

- `exposure bias`
- `sequence-level mismatch`

这部分最容易被误解的地方是：

- 训练目标已经很合理
- 但推理阶段的问题并不会自动消失

因为训练时模型看到的是“正确历史”，而推理时模型只能依赖“自己刚刚生成的历史”。

## 为什么 MLE 本身并没有错

`maximum likelihood estimation` 并不是坏目标。

它的问题不在于“完全没用”，而在于：

- 它优化的是局部下一个 token 概率
- 人类更在意的却是整段文本的整体质量

所以课程强调的是“目标之间有错位”，而不是简单否定 `MLE`。

## teacher forcing

课件里明确点到了 `teacher forcing`。

它的含义是：

- 训练时每一步都把真实前文喂给模型
- 而不是喂模型上一步自己的输出

这样训练更稳定，但也导致训练和推理阶段条件分布不一致。

它之所以常用，是因为：

- 优化更容易
- 梯度信号更稳定
- 模型更容易学会局部条件分布

所以它不是错误设计，而是一个有明显副作用的有效训练近似。

## exposure bias

`exposure bias` 指的是：

- 训练时模型总在理想前文上学习
- 测试时它必须在自己可能犯错的前文上继续生成

一旦早期生成出错，后续错误就会逐步累积。

生成质量常常会出现一种典型现象：

- 开头还不错
- 越往后越容易发散、重复或跑偏

因为后面的每一步都建立在前面已经可能出错的上下文上。

## greedy decoding

最简单的解码方式是每一步都选概率最大的 token。

优点：

- 简单
- 快
- 可复现

缺点：

- 容易局部最优
- 容易保守和重复

它的问题不在于“总是错”，而在于过于贪心地相信当前局部最优选择会导向整体最优输出。

但在开放式生成里：

- 当前最可能的下一个词
- 不一定通向最自然、最有趣或最有信息量的整段文本

## greedy 为什么在某些任务里仍然能用

如果任务本身更确定、输出空间更受限，那么 greedy 有时仍然是可接受的。

例如：

- 某些结构化生成
- 对稳定性要求高的场景

因此解码策略并不是永远有绝对优劣，而是和任务类型相关。

## beam search

`beam search` 会保留多条高分候选路径，而不是只保留一条。

它在条件生成任务里常见，例如：

- machine translation
- summarization

但在开放式生成里，较大的 `beam size` 并不一定更自然，反而可能更模板化。

它的核心思想是：

- 当前局部最优不一定等于整体最优
- 需要保留若干个可能继续发展的候选序列

所以 `beam search` 更像是一种有限宽度的搜索，而不是简单的逐步贪心。

## 为什么 beam search 在开放式生成里常不理想

因为开放式生成里“概率高”常常和“更安全、更常见、更无聊”的输出相关。

于是 `beam search` 可能会偏向：

- 套话
- 模板化
- 低风险但低信息量的输出

这也是现代开放生成更常依赖采样策略的原因。

## sampling

如果希望输出更有多样性，常见做法是采样。

常见控制手段包括：

- `temperature`
- `top-k`
- `top-p / nucleus`

采样的核心思想是：

- 不把模型概率分布当成一个只取 argmax 的工具
- 而是把它当成一个候选空间，从中按某种规则抽取输出

这样做的目标是提升：

- 多样性
- 自然度
- 不同候选之间的变化空间

### temperature

温度越低，分布越尖锐，输出越保守；
温度越高，输出越发散。

从直觉上看，它是在控制：

- 模型愿意多大胆地探索低于 argmax 的候选

### top-k

只在最高概率的前 `k` 个 token 里采样。

它的意义是：

- 砍掉特别离谱的低概率词
- 同时保留一定随机性

### top-p

保留累计概率达到阈值 `p` 的最小候选集合，因此候选规模会随分布动态变化。

这也是它比固定 `top-k` 更灵活的地方：

- 如果分布很尖，就保留很少候选
- 如果分布较平，就自动保留更多候选

因此它常被认为更适合作为开放式生成的默认策略之一。

## degeneration

高概率序列不一定是高质量序列。

课程里强调的常见退化现象包括：

- 重复
- 套话
- 空洞
- 局部合理但整体质量下降

这里的核心认识是：

- 模型最擅长的是复现训练语料里的高概率模式
- 但人类偏好往往不只取决于高概率

人类还会在意：

- 信息量
- 新鲜度
- 连贯性
- 任务符合度

因此“高概率”与“高质量”并不等价。

## sequence-level training

如果只优化 `MLE`，模型学到的是 token-level 目标。

但实际生成中，人们更在意的是：

- `BLEU`
- `ROUGE`
- 人类偏好
- 整体任务完成度

因此会出现：

- `policy gradient`
- `reinforcement learning`
- `self-critical sequence training`

这类方法的共同点是：

- 把整段输出当成优化单位
- 尽量让训练目标更贴近最终使用指标

模型不只是学“每一步接什么词”，还尝试学“整段输出最后会不会被判得更好”。

## 为什么 sequence-level training 会更难

因为序列级奖励通常：

- 更稀疏
- 更高方差
- 更难优化

所以这类方法常常不是直接替代 `MLE`，而是和 `MLE` 结合使用。

## scheduled sampling

课件把它作为缓解训练和推理错位的一种思路。

它的大致做法是：

- 训练时不总是喂真实前文
- 有时也喂模型自己生成的前文

这样模型会逐渐适应“自己给自己提供上下文”的情况。

它是在训练阶段人为制造一部分推理环境，让模型不总停留在“完美真实前文”的条件下。

## 生成评测为什么难

开放式生成往往有多个合理答案，因此：

- 只看词面重合不够
- 只看 `perplexity` 也不够

现代生成评测通常需要结合：

- 自动指标
- 人工评测
- 偏好比较
- factuality / robustness 检查

这部分最重要的判断标准不是“有没有万能指标”，而是：

- 不同生成任务需要不同评测组合

例如：

- 翻译更适合相对固定的参考对比
- 开放式写作更依赖人工评测和偏好比较
- 事实性任务还要额外检查 hallucination 风险

## 从课程主线看这一章的重要性

前面几章更多在讲“模型怎么得到表示和能力”。

这一章开始强调：

- 模型最终是要输出文本给人用的

因此训练目标、推理算法和评测标准必须一起看，不能只看其中一项。

## 需要记住的结论

- `teacher forcing` 让训练稳定，但会引出 `exposure bias`。
- `greedy`、`beam search`、`sampling` 会明显改变输出风格。
- 开放式生成常更依赖 `top-k / top-p` 等采样策略。
- `MLE` 与最终使用目标之间存在 `sequence-level mismatch`。
- 文本生成不是“训练完就结束”，解码和评测本身也是系统设计的一部分。
