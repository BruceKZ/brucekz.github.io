# Pretraining and Transfer Learning / 预训练与迁移学习

Covered in: Week 4 (`10-PLMs_1`, `11-PLMs_2`)

这一部分讨论的是现代 NLP 的核心范式转变：从“为每个任务单独设计模型”转向“先预训练一个大模型，再迁移到很多任务”。

## 为什么会从静态词向量走到预训练模型

静态词向量有两个明显限制：

- 同一个词在不同上下文里只有一个固定向量
- 迁移时通常只迁移局部表示，而不是整套模型能力

因此后续方法开始把“上下文化表示”和“完整模型迁移”放到中心位置。

这一步变化的根源有两个：

- 语言理解需要上下文敏感表示
- 下游有标签数据通常不足以从零学出强模型

因此更自然的思路变成：

- 先在大规模无标签语料上学通用参数
- 再把这些参数迁移到具体任务

## 预训练到底在学什么

预训练不是在学某一个特定任务标签，而是在学：

- 语言分布
- 共现结构
- 语义与句法模式
- 广泛的世界知识和文本规律

这让模型在进入下游任务前，就已经拥有了一个很强的初始化状态。

## ELMo

`ELMo` 用双向 `LSTM` 产生上下文化词表示。

它的关键改进是：

- 同一个词在不同上下文里会有不同表示
- 多层表示可以按任务加权组合

它仍然有一个重要局限：

- 双向性本质上仍然是前向和后向语言模型的拼接
- 不是原生统一双向编码

`ELMo` 的历史位置正好处在两个阶段之间：

- 它已经超越了静态词向量
- 但还没有进入后面 `BERT` 那种统一双向 transformer 编码

因此它通常被看作现代上下文化表示的重要过渡点。

## 为什么上下文化表示会比静态词向量强

因为很多语言现象只有在上下文里才能确定。

例如：

- 一个词当前是名词还是动词
- 一个多义词当前是哪种意义
- 某个词对整句作用是主题、修饰还是补充

静态词向量无法根据上下文改变自身表示，而上下文化表示可以。

## BERT

`BERT` 是 `encoder-only transformer`，主要依靠 `masked language modeling` 做预训练。

它的典型训练设置包括：

- 选取 15% 的 token 作为预测目标
- 其中 80% 替换为 `[MASK]`
- 10% 替换成随机词
- 10% 保持原样

这样做的效果是：

- 预训练时能同时利用左右上下文
- 对分类和序列标注特别有效

这个设计最关键的地方在于：

- 模型不是只看左边去猜下一个词
- 而是能在被遮住位置同时利用左右信息

因此 `BERT` 天然更像一个强编码器，而不是纯生成器。

## 为什么不是把所有被选中的 token 都替换成 `[MASK]`

课件里给出的 80/10/10 规则，就是在缓解一个训练和使用场景不完全一致的问题。

如果训练时总是看到大量 `[MASK]`，而下游使用时几乎看不到 `[MASK]`，模型会产生分布差异。

保留一部分随机词和原词，可以让预训练输入更接近真实文本分布。

## `[CLS]` 与 fine-tuning

做分类时，常见做法是在序列开头加入 `[CLS]`，再用它的最终表示接分类头。

课里强调的结论是：

- 完整 `fine-tuning` 通常比只冻结 BERT 表示再外接模型更强

这说明现代迁移学习真正迁移的不是“固定特征”，而是：

- 整个参数空间里的通用能力

模型会在下游任务监督下进一步调整内部表示，因此通常比只把它当静态特征抽取器更有效。

## BERT 更擅长什么

从课程主线看，`BERT` 更擅长的是：

- classification
- sequence labeling
- sentence-pair understanding

这些任务都更依赖强编码表示，而不需要模型自然地一步步自回归生成长文本。

## GPT

`GPT` 走的是 `decoder-only transformer` 路线，预训练目标是 `causal language modeling`。

特点是：

- 只能看左上下文
- 天然适合自回归生成

和 `BERT` 的常见对比是：

- `BERT` 更偏编码和理解
- `GPT` 更偏生成

这里不要把“更偏生成”理解成“完全不能做别的任务”，而是说它的预训练方式更自然地支持：

- 续写
- 条件生成
- 长文本自回归建模

因此后来大模型路线更多沿着 `GPT` 风格演化，并不奇怪。

## 为什么 BERT 和 GPT 会形成两条典型路线

因为它们解决的是两类不同偏好的问题：

- 如果更重视双向表示学习，就偏向 `BERT`
- 如果更重视自然生成和统一接口，就偏向 `GPT`

后面很多模型，其实都可以放到这两条路线和它们的混合体之间来理解。

## BART

`BART` 是 `encoder-decoder transformer`。

它的思路是把两类优势结合起来：

- `encoder` 处理被破坏过的输入
- `decoder` 自回归地重建输出

课件里强调的典型 corruption 包括：

- `text infilling`
- `sentence permutation`

因此 `BART` 的一个关键特点是：

- encoder 负责理解被扰动的输入
- decoder 负责把它恢复成合理输出

这使它天然既能做理解相关任务，也能做生成相关任务。

## BART 为什么适合生成任务

因为它本来就有完整 decoder。

相比 `BERT` 这类纯编码器模型，它在做：

- summarization
- generation
- translation

这类任务时结构上更自然。

## T5

`T5` 的核心观念不是单纯一个结构，而是统一任务接口：

- 把各种任务都改写成 `text-to-text`

这样分类、问答、摘要都可以写成统一的输入输出形式。

这一点的意义在于，它把很多原本看起来不同的 NLP 任务放进了同一个范式：

- 输入是文本
- 输出也是文本

从系统设计角度看，这会让训练、推理和任务适配都更统一。

## T5 的价值不只在模型，还在任务表述

课程里需要抓住的是：

- `T5` 不是简单换了个结构名字
- 它把“任务如何表达给模型”也系统化了

这一点对后面的 instruction tuning 和大模型接口设计有直接影响。

## RoBERTa 与 DistilBERT

这两点在课件里出现过，复习时不应漏掉。

### RoBERTa

可以理解成对 `BERT` 训练策略的强化版本：

- 数据更多
- 训练更充分
- 性能通常更强

### DistilBERT

它是通过知识蒸馏得到的更小模型：

- 参数更少
- 推理更快
- 性能损失相对有限

这两个补点的意义在于，它们说明：

- 预训练模型发展不只靠全新架构
- 也靠训练策略优化和模型压缩

因此现代模型进步既可能来自结构创新，也可能来自训练和部署层面的工程改进。

## 预训练与迁移学习的范式变化

旧范式通常是：

- 先做任务特定模型
- 最多拿词向量做初始化

新范式通常是：

- 先在海量文本上做预训练
- 再把整套模型迁移到很多下游任务

这是现代 NLP 的关键转折。

这一转折背后的核心判断是：

- 通用语言能力可以先学
- 下游任务不必从零开始

这直接改变了研究和工程实践：

- 数据更大
- 模型更大
- 训练更集中在预训练阶段
- 下游任务更多依赖适配与 fine-tuning

## 从课程主线看本章重点

最重要的不是死记每个模型年份，而是要会回答：

1. 它是 `encoder-only`、`decoder-only` 还是 `encoder-decoder`
2. 它的预训练目标是什么
3. 它更自然地适合哪类任务
4. 它相对前一代模型解决了什么问题

只要这四个问题能答清楚，整章结构就不会乱。

## 需要记住的结论

- `ELMo` 引入了上下文化表示，但还不是统一双向编码。
- `BERT` 是 `encoder-only + masked LM`。
- `GPT` 是 `decoder-only + causal LM`。
- `BART` 和 `T5` 属于 `encoder-decoder` 预训练路线。
- `RoBERTa` 和 `DistilBERT` 是课程里出现过的重要补充点。
- 这部分的核心是“迁移整套模型能力”，而不只是迁移词向量。
