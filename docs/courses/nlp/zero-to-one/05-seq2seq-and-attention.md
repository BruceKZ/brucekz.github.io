# Seq2Seq and Attention / Seq2Seq 与注意力

Covered in: Week 3 (`7-Seq2seq`)

这一部分的主线是：当任务从“输入一句话，输出一个标签”变成“输入一个序列，输出另一个序列”时，基础分类模板已经不够，需要新的结构。

## 什么是 Seq2Seq

`sequence-to-sequence` 任务的特点是：

- 输入是一个序列
- 输出也是一个序列

典型任务包括：

- machine translation
- summarization
- dialogue

和文本分类相比，这类任务的难点在于：

- 输出长度不固定
- 输出不是从一个有限标签集合里选，而是一步步生成
- 输入和输出之间往往存在复杂对齐关系

因此“句子表示 + 分类头”这种结构已经不够用。

## encoder-decoder 结构

最经典的 `seq2seq` 模型由两部分组成：

- `encoder`：读取输入序列
- `decoder`：条件生成输出序列

训练目标通常仍然是条件语言模型，也就是在给定输入和前面目标词的条件下，预测下一个目标词。

- `encoder` 负责把输入压成可供后续使用的表示
- `decoder` 负责在这些条件下逐步生成输出

因此 `seq2seq` 并没有离开课程前面的主线，它只是把输出空间从标签变成了序列。

## decoder 为什么天然像语言模型

`decoder` 在每一步都要根据：

- 输入条件
- 之前已经生成的目标词

去预测下一个目标词。

因此它是一个条件语言模型。

这点非常重要，因为后面的生成模型、翻译模型和现代大模型都能继续放进这个框架理解。

## temporal bottleneck

早期 `encoder-decoder` 模型常把整个输入序列压成一个固定长度向量，再交给 `decoder` 使用。

这会产生 `temporal bottleneck`：

- 输入越长
- 固定向量需要压缩的信息越多
- 信息损失越明显

这里的问题不只在于“信息少了一点”，而是整个信息流被迫经过一个单点。

如果输入句子很长，或者输入输出之间有复杂对应关系，那么让一个固定向量同时保留：

- 主题信息
- 词义信息
- 顺序信息
- 局部对齐线索

会变得非常困难。

## 为什么 Seq2Seq 特别容易暴露这个瓶颈

在分类任务里，输出只有一个标签。

但在翻译、摘要里，decoder 每一步都可能需要输入序列中不同部分的信息：

- 翻译某个词时，需要找到对应输入位置
- 生成摘要某一段时，需要定位输入里的相关片段

如果所有这些都只靠同一个固定向量承载，就会很吃力。

## attention 的基本思想

`attention` 的引入，是为了解决固定瓶颈问题。

它的核心做法是：

- `decoder` 每一步生成时
- 不只依赖一个固定句向量
- 而是回头去看 `encoder` 在不同位置的隐藏状态

这样模型就能为不同输入位置分配不同权重。

这相当于把原本“所有输入信息先挤成一个点”的结构，改成：

- 保留输入各位置的表示
- decoder 每一步按需读取

因此信息流会灵活得多。

## attention 在做什么

它通常包含三步：

1. 计算当前解码状态与各个编码状态的相似度
2. 用 `softmax` 把这些分数变成权重
3. 用这些权重对编码状态做加权平均

这得到的加权结果会作为当前解码步骤的条件信息。

这里不必先背公式，先抓住两层直觉：

- 当前 decoder 状态会决定“现在更该看输入哪里”
- attention 权重会决定“这些输入位置各自贡献多少”

因此 attention 的结果不是单纯查表，而是一次针对当前生成步骤的动态信息汇总。

## Query / Key / Value 的直观理解

为了统一后面的 `transformer` 记法，可以先这样理解：

- `query`：当前需要找什么信息
- `key`：每个位置拿什么标签参与匹配
- `value`：每个位置真正提供什么内容

这个视角会在后面的自注意力中继续出现。

虽然早期 `encoder-decoder attention` 不一定总显式写成 `Q/K/V` 形式，但这个视角很好用，因为它能把很多注意力机制放进统一框架。

## attention 权重到底代表什么

初学时很容易把 attention 权重理解成“模型真实在看哪里”。

更准确地说：

- 它表示当前步骤下，各输入位置对当前输出表示的相对贡献

它常常和对齐有关，但不一定等于严格的人类语言学对齐。

## attention 为什么能缓解长距离依赖

因为它把 decoder 和 encoder 各位置之间建立了直接连接。

在没有 attention 时，输入早期信息要先被编码到固定向量里，再一路传给 decoder。

有了 attention 以后：

- decoder 可以直接访问远处输入位置
- 不必完全依赖单一压缩向量

这会显著减轻长距离信息丢失问题。

## attention 的作用

它至少带来三点改进：

- 减轻固定长度瓶颈
- 让长距离依赖更容易被访问
- 产生某种对齐关系，便于模型定位输入中相关位置

再进一步说，它还带来一个课程里非常重要的结构变化：

- 编码和解码之间的信息交换，不再只有一个固定接口
- 而是变成了逐步、动态、位置感知的交互

因此 `attention` 被视为从早期 `seq2seq` 到现代 `transformer` 的关键过渡点。

## 训练时和生成时的差别

和其他生成模型一样，`seq2seq` 在训练时通常也依赖真实历史目标词来进行监督。

这说明：

- 训练阶段更稳定
- 推理阶段仍然需要模型自己逐步生成

因此后面文本生成一章里讲的很多问题，例如：

- decoding
- exposure bias

这一点也为后面的模型结构铺垫了方向。

## 这一节还没有彻底解决什么

即使加入了 `attention`，如果底层 `encoder / decoder` 仍然是 `RNN`，那么：

- 计算仍然是串行的
- 长序列训练和推理仍然较慢

这也是后续 `transformer` 的直接背景。

attention 解决的是“信息如何更合理地流动”，但还没有解决“模型如何并行、如何更高效计算”的问题。

因此课程到下一步时，关注点会转向：

- 能不能保留注意力的优势
- 同时去掉循环结构本身

## 需要记住的结论

- `Seq2Seq` 用于输入输出都是序列的任务。
- 早期 `encoder-decoder` 会遇到 `temporal bottleneck`。
- `attention` 允许 `decoder` 在每一步直接访问输入各位置状态。
- `Query / Key / Value` 是理解后续 `transformer` 的统一视角。
- `attention` 不是在“替代 decoder”，而是在给 decoder 增加一个按需读取输入信息的动态接口。
