# Text Generation / 文本生成

Covered in: Week 6, Lectures 19-21 (`19+20-Text_Generation_part1`)

这里的重点是：语言模型不只是会“预测下一个 token”，还要在生成阶段做一连串 decoding 决策，而这些决策会直接影响文本质量、长度、多样性和可控性。

## 训练目标和生成目标并不完全一致

大多数生成模型的基础训练目标还是 `maximum likelihood estimation`，也就是让模型最大化训练语料上真实下一个词的概率。

这一步学到的是：

- 局部条件概率
- 大规模语言规律
- 如何继续一段文本

但真正生成时，模型要自己不断把先前生成的 token 再喂回去。
这就带来两个典型问题：

- **exposure bias：** 训练时看到的前文是真实文本，测试时看到的前文部分来自自己
- **sequence-level mismatch：** 训练优化的是逐 token 概率，使用者关心的却是整句、整段甚至整篇文本质量

## 贪心解码 greedy decoding

最直接的做法是每一步都选概率最大的 token。

它的优点是：

- 简单
- 快
- 可重复

缺点也很明显：

- 容易陷入局部最优
- 文本往往很保守
- 容易重复

所以 `greedy decoding` 虽然适合某些确定性任务，但通常不是开放式生成里最好的默认选项。

## Beam search

`beam search` 会在每一步保留若干条高分候选序列，而不是只保留一条。

它的直觉是：

- 单步最优不一定等于整句最优
- 多保留几条路径，后面可能找到更好的整体序列

`beam search` 常见于：

- machine translation
- summarization
- 其他相对目标明确的条件生成任务

但课上的一个核心提醒是：更大的 `beam size` 不一定让开放式文本更自然。
在开放式生成里，它反而可能把模型推向高概率但无聊、模板化甚至重复的输出。

## Sampling

如果想让模型输出更有多样性，常见办法是 `sampling`，也就是按概率分布随机采样，而不是永远拿最大值。

这类方法的关键控制参数包括：

- `temperature`
- `top-k sampling`
- `top-p` / `nucleus sampling`

### Temperature

`temperature` 会改变分布的尖锐程度。

- 温度低：更保守，更接近贪心
- 温度高：更发散，随机性更强

### Top-k sampling

`top-k` 先把候选空间限制在概率最高的前 `k` 个 token，再在里面采样。

它的作用是：

- 去掉很离谱的低概率词
- 保留一定随机性

### Top-p / nucleus sampling

`top-p` 不是固定保留多少个 token，而是保留累计概率达到阈值 `p` 的最小集合。

它的直觉更灵活：

- 分布很尖时，候选集合可以很小
- 分布很平时，候选集合可以自动变大

这也是现代开放式生成里非常常见的默认方案之一。

## 退化问题 degeneration

课程里一个高频结论是：高概率解码不等于高质量文本。

语言模型常见的 `degeneration` 现象包括：

- 重复循环
- 空洞套话
- 过于机械的安全延续
- 局部合理但全局越来越偏

这说明最大化似然得到的模型，在直接最大化序列概率时，并不天然等于人类偏好的文本生成器。

## 条件生成和控制

很多生成任务不是“自由续写”，而是带条件的：

- 根据输入句子翻译
- 根据文档摘要
- 根据问题生成回答
- 按指定风格生成文本

这时候模型不仅要 fluent，还要：

- 忠实于输入
- 满足任务约束
- 控制长度和内容范围

于是 decoding 就不只是“生成最像自然语言的文本”，还要兼顾任务约束。

## 训练改进：从 token-level 到 sequence-level

如果只用 `MLE`，模型优化的是局部 token 预测。
但很多真正关心的目标是序列级别的，比如：

- `BLEU`
- `ROUGE`
- 任务成功率
- 人类偏好

因此会有一些 `sequence-level training` 方法，把整段输出作为优化对象。

典型思路包括：

- `policy gradient`
- `reinforcement learning`
- `self-critical sequence training`

它们的目标是直接优化更接近最终使用指标的信号，而不是只优化下一个 token 的条件概率。

## 评测文本生成

生成评测一直很难，因为开放式文本常常存在多个正确答案。

传统做法包括：

- `BLEU`
- `ROUGE`
- `METEOR`

它们方便、便宜，但局限也非常明显：

- 过度依赖参考答案表面重合
- 容易误伤合理改写
- 很难覆盖创造性和事实性

所以现代生成评测通常还要结合：

- 人工评测
- 偏好比较
- 任务完成度
- 鲁棒性和事实性检查

## 核心概念

- **训练/推理错位：** `MLE` 优化的是局部 token 概率，生成阶段关心的是整段文本质量。
- **解码方法：** `greedy`、`beam search`、`sampling` 会显著改变输出风格和错误类型。
- **序列级优化：** 许多生成任务会引入 `sequence-level training` 来直接对齐最终目标。

## 高频结论

- **高概率不等于高质量：** 最大概率序列可能非常无聊，甚至更容易出现重复和退化。
- **开放式任务更依赖采样：** `top-k` 和 `top-p` 往往比纯 `beam search` 更适合开放生成。
- **评测不能只看词面重合：** 开放式生成需要多种指标和人工判断联合评估。

## 小结

文本生成的难点，不只在模型会不会“接下一词”，还在于如何把局部概率模型变成全局上更自然、更有用、更可控的输出系统。训练目标、解码策略和评测方式，三者必须一起看。
